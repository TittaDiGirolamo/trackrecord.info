# Contributing to Trackrecord Design Principles

Thank you for your interest in improving these principles.  
All changes should align with Trackrecord’s core values of **transparency**, **evidence**, and **respect for the user**.

## How to Propose Changes

1. **Open an Issue First** (recommended)
   - Go to the Issues tab of this repository.
   - Use the title format: `Proposal: [Short description of change]`
   - In the description, please include:
     - Which principle(s) the change affects
     - Clear rationale supported by evidence (user feedback, metrics, accessibility standards, performance data, etc.)
     - Expected impact on users or the product

2. **Create a Pull Request**
   - Create a new branch from `main`
   - Make your changes to `DESIGN-PRINCIPLES.md`
   - Update the **Version** number and **Last Updated** date in the header
   - Submit a Pull Request with a clear description of the change

## Review Criteria

Proposed changes will be evaluated on:
- Alignment with the Overall Philosophy
- Improvement in clarity or simplicity (Principle 3)
- Strength of supporting evidence
- Backward compatibility with existing principles
- Ethical implications

## Questions?

Open an issue with the `question` label.
